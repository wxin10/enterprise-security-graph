# 演示沙盘使用与答辩讲解手册

## 1. 导读目标
本手册是系统在完成部署后，为“验收交付演示”与“毕业设计答辩展示”环节专门编写的最佳实践路径指南。
我们已经在系统 `data/sample/` 目录下预埋了一套代表企业实战环境下最具典型性的内网渗透攻击链（APT 早期阶段）。通过阅读本文档，使用者可以迅速拉起包含真实攻防对抗时序的模拟数据大屏，以此检验并证明【基于图谱的分析】模式对于复杂安全事件独到的溯源解释力。

## 2. 样本背景故事线：一次典型的凭证窃取与内网横向扩散
我们在预制的数据包中复刻了一条发生于 2026 年 4 月的情景案例：

- **09:04 - 09:08 (资产嗅探与密码喷洒)**：
  不法分子携带恶意代理 IP (`203.0.113.42`) 对企业防火墙暴露的 VPN 网关进行端口探测并尝试暴破 `admin`, `test1`, `wangwei` 等内部账号。这些异常行为均被登录日志记录 (`RAW_L001-L004`)。由于触发了系统基于“同源IP短时多账号错误登录”特征，自动抛出了「外部源暴力破解告警」(`AL_001`)。

- **09:12 (突破防线)**：
  由于弱口令配置漏洞，黑客在最后关头通过撞库成功获取到了高级工程师 `lishi` 的访问凭据（`RAW_L005`），建立了核心业务网的隧道连接。

- **09:15 - 09:18 (内网情报收集与跨域横向渗透)**：
  黑客随即利用被盗账号进入内网横行，首先利用 `SMB` 协议大面积违规拖取代码核心仓库资料（`RAW_H001`）抛出高频调用告警（`AL_002`），随后企图以跳板形式登入核心承载数据库 `DB-SRV-CORE` 及中央域控服务器 `DC-SRV-01` (`RAW_H002, RAW_H003`)，因为属于违背管理基线的敏感跳跃，随即触发极其严重的【跨界横向访问】图模式告警 (`AL_003`)。

- **09:19 (全网协同熔断)**：
  自动防护体系根据图谱分析研判，将多项风险指标聚合，直接判定该特征高度吻合账号失窃特征。在不需要安全专家通宵人工干预的前提下，立刻发起了对于原始外联恶意源 `203.0.113.42` 的拦截熔断动作体系 (`BA_DEMO_01`)。

## 3. 一键挂载沙盘环境
请确保你的环境已经启动了 Neo4j 图数据库，然后在项目的根目录下执行如下集成命令：

```powershell
python scripts/bootstrap_demo_data.py
```

终端进度将依次滚动过“加载样本”、“数据清洗”、“图谱实体认知抽取”与“Neo4j 高效图库灌装”等四项主要步骤。期间不存在任何“黑盒”造数操作，每一个步骤都可以通过对应的 `.csv` 标准化数据集进行反求追查，完全符合企业级的可靠性原则。

## 4. 推荐的答辩/验收动线
推荐评审视角按照如下循序渐进的过程展示系统成果：

1. **首页(Dashboard) —— 宏观防御力佐证**
   - **展示项**：系统整体拦截数增加，并自动生成了最新的高风险资产卡片与恶意行为趋势。
   - **说辞匹配**：直接通过监控台确认“刚才导入了实战数据后，工作台马上就能全局透视出当前网络正遭遇一波严峻的多阶段攻击和撞库行动，而不需要翻看黑白界面数万行混杂的零散日志。”

2. **告警中心(AlertsView) —— 分析研判提效**
   - **展示项**：列表首屏可以看到 `Critical`（致命风险）等级的“内网跨域横向移动告警”。
   - **说辞匹配**：证明图计算策略相较于普通规则具有对高隐藏性的低频风险（横向扩赛）的深度识别捕获优势。

3. **图谱核心视图应用 —— 因果溯源还原**
   - **展示项**：切换点击查看涉事外联 IP `203.0.113.42` 或受害账号 `lishi` 相关的分析图谱。
   - **说辞匹配**：“从展示的图中，大家不仅能一目了然看清黑客最初用什么假账号敲门、拿到了哪台特定服务器授权，甚至能把最后的自动化封禁阻断动作 (`BA_DEMO_01`) 关联到一起，形成了真正的时序化侦测打击闭环证据链。”

4. **安全管控执行(BansView / ActionsView) —— 行动响应闭环**
   - **展示项**：在自动化惩戒管控台核查当前已经被成功踢掉和列名阻断的罪魁祸首实体 IP，或为被盗员工提出安全隔离处置审批单留档。
   - **可验证的量化优势**：这一流程如果在常规大厂环境中需要“取证、评估、走OA审批流程、上机配置”至少花费 4~24 小时，而在目前的“日志->图谱计算”机制协助下只需等待模型完成数秒计算即可一发阻拦。

## 5. 用于扩展研究的实验评估指标
借助此批样本驱动图谱数据库生成的知识集，你可以作为佐证填入论文当中的系统可行性分析或是效果对比实验图表中：
- **原始告警收敛率/压缩比**：多条独立的杂乱登录失效日志最终仅产生极少数的明确特征告警（例如用图结构追踪异常跳板比传统 IDS 流浪识别误报率低）。
- **处置敏捷度（MTTR，Mean Time to Respond）**：通过打通底层阻断图元，时间跨度从黑客初次扫描（09:04）直至最后一发跨级拦截（09:19）实现了防御纵深全记录，并缩减了防守方的响应差值。
- **孤立安全事件孤岛关联能力**：样本库虽然只提供 8 条结构简易、字段缺失的基础碎片，最终经过提取后能够长出超过 15 个具备立体交叉从属关联图拓扑结构网。
