# AGENTS.md

你正在协助开发一个毕业设计项目：

《基于 Neo4j 的企业网络恶意行为识别与封禁系统的设计与实现》

## 项目目标
1. 构建企业网络安全行为图谱
2. 基于日志识别恶意行为
3. 提供风险评分、告警展示、攻击关系追踪
4. 提供自动封禁和审计记录能力
5. 系统结果便于论文撰写和答辩演示

## 技术栈
- 后端：Python + Flask
- 前端：Vue3 + Element Plus + ECharts
- 图数据库：Neo4j
- 数据格式：CSV / JSON
- 接口风格：RESTful API

## 编码规则
1. 所有代码必须完整，不要只给片段
2. 所有代码必须带中文详细注释
3. 新增代码时必须说明文件路径
4. 优先保证代码可运行，再逐步扩展复杂功能
5. 优先采用工程化目录结构
6. 对版本兼容、跨域、路径、Neo4j 驱动等问题要主动提醒
7. 后端接口统一返回 JSON 结构
8. 前端页面风格应接近企业安全平台
9. 输出内容尽量包含可用于论文撰写的技术说明

## 工作方式
1. 一次只完成一个模块
2. 先说明思路，再生成代码
3. 不要一次生成整个系统
4. 如果任务涉及多个文件，必须列出新增和修改的文件清单
5. 每轮结束时，给出运行方法和下一步建议

## 项目目录约定
- docs/ 放系统设计、任务计划、接口文档、测试文档
- data/raw/ 放原始日志样例
- data/processed/ 放清洗后的实体与关系数据
- neo4j/ 放 schema、导入和查询相关 Cypher
- scripts/ 放数据清洗、抽取、导入、识别和封禁脚本
- backend/ 放 Flask 后端
- frontend/ 放 Vue3 前端
- tests/ 放测试脚本和测试说明
- deploy/ 放启动说明和部署文件
